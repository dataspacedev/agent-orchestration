<objective>
Build the registry-announcer — a standalone Go binary that runs as a sidecar in every agent pod. It fetches the agent's AgentCard, writes the AgentDirectory CR, satisfies the pod ReadinessGate, and queues a background write to inventory-api.

Purpose: This is what makes agents visible to the platform. Without a running announcer, AgentDirectory entries are empty stubs and pods never become Ready.
Output: New registry-announcer/ module with cmd/main.go, internal/announcer package, tests, Dockerfile, and Makefile.
</objective>

    registry-announcer/internal/announcer/announcer.go:

    Struct Announcer with fields:
    - k8sClient (controller-runtime client.Client)
    - podName, podNamespace, agentName, agentURL, inventoryAPIURL string

    Method Announce(ctx context.Context) error:
    1. HTTP GET to agentURL + "/.well-known/agent.json"; if fails, return error
    2. Read raw response bytes; compute SHA-256 hash as hex string
    3. Fetch existing AgentDirectory CR (get by name=agentName, namespace=podNamespace) using unstructured client.Get; if 404 that is an error (controller must pre-create it)
    4. Compare existing cardHash field to computed hash: if equal, skip AgentDirectory patch (idempotency)
    5. If different (or empty): build patch map with spec fields (agentName, version, url, skills as []string of skill IDs, cardHash, readyAt=now); apply via client.Patch with types.MergePatchType
    6. Patch pod status condition: strategic merge patch on pod/status to set agents.orchestration.io/directory-ready=True
    7. Start background goroutine for secondary write to inventory-api (if inventoryAPIURL != ""): POST /agents with agent card JSON; on error, log and retry with exponential backoff using wait.ExponentialBackoffWithContext (max 5 retries, 2s base, 2x multiplier); goroutine runs until ctx cancelled
    8. Return nil (secondary write result does NOT affect return value)
